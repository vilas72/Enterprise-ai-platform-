# Architecture Decision Records

ADRs capture important technical decisions for the Enterprise AI Platform.

## Naming

Use `ADR-NNNN-DecisionName.md`, where `NNNN` is the next sequential number.

## Status

Use one of: Proposed, Accepted, Superseded, or Deprecated. When a decision
changes, create a new ADR and link both records instead of rewriting history.

## Required Structure

Every ADR must include all sections from [template.md](template.md), even when
a section currently contains "None."
